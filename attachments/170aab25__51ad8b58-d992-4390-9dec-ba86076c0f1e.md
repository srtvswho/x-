# 信号源记分牌系统 设计文档 v0.1

> 目标:系统化地发现、验证、追踪"有真实研究能力的公开信号源"(推特/知乎/Reddit 等),
> 把非结构化的观点内容转化为可回测的预测记录,用数据决定信任谁、信任多少、何时降权。
> 本文档可直接作为开发规格交给 Claude Code 实现。

---

## 0. 设计原则

1. **只采信可证伪的预测**:有时间戳、有具体标的、事后可验证。忽略一切自报收益率。
2. **研究分与影响力分分离**:前者衡量"判断是否正确",后者衡量"发声是否移动价格"。两者用法完全不同。
3. **预测记录(PredictionRecord)是系统的原子单位**:所有评分都从它聚合而来,源本身只是预测记录的容器。
4. **原始内容必须存档**:删帖行为本身是重要信号,系统必须能发现"他删了当初的错误预测"。
5. **防小样本造神**:3 次命中不等于 90% 胜率,所有评分必须带不确定性度量。

## 1. 核心数据结构

### 1.1 Source(信号源)

```json
{
  "source_id": "tw_aleabitoreddit",
  "platform": "twitter | zhihu | reddit | substack | xueqiu",
  "handle": "@aleabitoreddit",
  "display_name": "Serenity",
  "first_seen_at": "2025-07-01",
  "first_tracked_at": "2026-06-10",
  "discovery_channel": "outcome_tracing | graph | content_fingerprint | cross_platform | manual",
  "domain_tags": ["AI供应链", "光通信", "半导体小盘"],
  "status": "candidate | watchlist | active | demoted | blacklisted",
  "follower_count_history": [{"date": "...", "count": 50000}],
  "scores": "→ SourceScore",
  "red_flags": ["→ RedFlag[]"],
  "notes": "人工备注"
}
```

### 1.2 PredictionRecord(预测记录)— 原子单位

```json
{
  "prediction_id": "uuid",
  "source_id": "tw_aleabitoreddit",
  "published_at": "2026-01-12T14:30:00Z",
  "captured_at": "2026-01-12T14:42:00Z",
  "raw_url": "https://x.com/...",
  "raw_text": "原文全文",
  "archive_url": "本地存档/archive.org 快照",
  "is_deleted": false,

  "ticker": "RPI.L",
  "market": "US | A股 | HK | JP | crypto",
  "direction": "long | short | neutral",

  "claim_type": "quantitative | directional | thematic",
  "quantitative_claim": {
    "metric": "fy_revenue_growth",
    "predicted_value": 0.55,
    "consensus_at_time": 0.14,
    "actual_value": null,
    "resolution_date": "2026-03-31"
  },

  "horizon": "1w | 1m | 3m | 6m | event_driven",
  "conviction": 3,
  "is_repeat_call": false,
  "repeat_of": null,
  "thesis_summary": "LLM 生成的一句话论点摘要",
  "thesis_category": "瓶颈/产能 | 份额提升 | 财报预期差 | 政策 | 叙事",

  "verification": "→ Verification"
}
```

字段说明:
- `published_at` vs `captured_at`:两者之差就是你的系统延迟,必须记录,回测时用 captured_at 计算可实现收益,避免回测虚高。
- `conviction`(1-5):由 LLM 评估。1=随口提及/转发,3=给出完整论据,5=明确表述为最高仓位/最强信念。
- `is_repeat_call`:同一源对同一标的的重复喊单要标记。重复喊单的边际信息为零,且密集重复喊单常出现在出货阶段。
- `claim_type` 三档:
  - quantitative:给出可量化预测值(营收、产能、份额)→ 最高质量,可与共识对比
  - directional:明确方向+标的+大致时间 → 可用价格验证
  - thematic:看好某赛道无具体标的 → 仅作叙事风向参考,不计入研究分

### 1.3 Verification(验证结果)

```json
{
  "status": "pending | verified_hit | verified_miss | expired | unverifiable",
  "price_returns": {
    "r_1w": 0.08, "r_1m": 0.31, "r_3m": 1.20,
    "benchmark_1w": 0.01, "benchmark_1m": 0.03, "benchmark_3m": 0.07,
    "sector_etf_1m": 0.05,
    "excess_1m": 0.28
  },
  "entry_price_basis": "captured_at 后下一个可成交价(开盘价/下一分钟均价),绝不用 published_at 时点价",
  "quantitative_outcome": {
    "actual": 0.58,
    "error_vs_prediction": 0.03,
    "error_vs_consensus": 0.44,
    "beat_consensus": true
  },
  "verified_at": "..."
}
```

关键规则:
- 超额收益基准用两层:大盘指数 + 行业 ETF。小盘 AI 股在 AI 牛市里涨 30% 可能纯是 beta。
- 入场价基准必须用 `captured_at` 之后的可成交价。对影响力大的源,发文后已跳涨的部分不属于你。
- `expired`:超过 horizon 仍无法判定的预测按平局处理,但计入"含糊预测率"。

### 1.4 SourceScore(源评分)

```json
{
  "research_score": {
    "n_predictions": 35,
    "n_verified": 31,
    "hit_rate": 0.74,
    "hit_rate_wilson_lower": 0.57,
    "avg_excess_return_1m": 0.12,
    "avg_excess_return_3m": 0.25,
    "quantitative_accuracy": "量化预测的平均误差 vs 共识误差",
    "specificity_avg": 3.8,
    "rolling_6m_hit_rate": 0.65
  },
  "influence_score": {
    "avg_price_reaction_1h": 0.06,
    "avg_price_reaction_1d": 0.11,
    "media_citations": 12,
    "follower_growth_30d": 8.5,
    "reaction_trend": "rising | stable | falling"
  },
  "composite_trust": "研究分的 Wilson 下界 × 衰减因子,用于仓位权重",
  "regime": "alpha_source | event_mover | decayed | suspect"
}
```

评分要点:
- **Wilson 下界(或贝叶斯收缩)是核心防过拟合手段**:10 中 9 的源,Wilson 95% 下界约 0.57;3 中 3 的源下界只有 0.31。用下界排序,自动惩罚小样本。
- **rolling 窗口检测衰减**:全历史胜率高但近 6 个月滚动胜率显著下滑 → 自动降权并标记 `decayed`。
- **regime 判定**(决定该源怎么用):
  - `alpha_source`:研究分高、影响力低 → 跟其逻辑做中期仓位,黄金状态
  - `event_mover`:影响力高(发文后 1h 反应 > 阈值)→ 其推文是事件而非信号,只能做事件交易或观察,禁止慢速傻多
  - `decayed`:滚动胜率衰减 → 降权观察
  - `suspect`:触发红旗 → 冻结

## 2. 发现管道:四条进料通道

### 通道 A:结果反推(优先实现)

逻辑:不扫人,扫异常事件,再反查谁提前预言了它。

```
Step 1  异动筛选器(每日/每周批处理)
        - 财报营收/EPS 超共识 > 25% 的中小盘股(市值 < 50亿美元)
        - 无重大公告但 20 交易日累计超额收益 > 40% 的小盘股
        - 数据源:财报日历 + 共识预期(可用免费源起步) + 日线行情
Step 2  对每个异动标的,搜索社交平台历史
        - 推特高级搜索: "$TICKER" OR "公司名" since:T-180d until:T-7d
        - Reddit/知乎/雪球同理
Step 3  LLM 判别每条历史讨论:
        - 是否在事前给出了方向 + 论据(产业逻辑/渠道调研/数据)
        - 排除:事后吹牛、无论据纯喊单、新闻转述
Step 4  命中者进入候选池,记录其该笔"已验证预测"作为种子记录
```

### 通道 B:聪明关注者图谱

```
Step 1  手工构建种子集:30-50 个确认有研究水平的账号
        (已验证的分析师、产业从业者、通道A捞出的已验证源)
Step 2  抓取种子集的关注列表(following)
Step 3  计算每个被关注账号的 smart_follower_ratio:
        被多少个种子账号关注 / 自身总粉丝数
Step 4  高比值 + 总粉丝 < 5万 的账号进入候选池
Step 5  补充:抓取已验证源出名前(粉丝<某阈值时期)的深度互动者
        (长回复、被本尊认真回复的账号)
```

### 通道 C:内容指纹

对候选池及关键词增量做 LLM 文本分类,正面指纹:
- 给出可证伪数字(营收/产能/份额预测)
- 引用一手信源(财报原文、专利、行业会议、产业链调研)
- 讨论无卖方覆盖的小市值标的
- 术语密度高、传播性差(出名前 alpha 源的典型形态)

负面指纹(直接过滤):
- 互动诱饵(抽奖、关注转发)、晒单截图为主、自报收益率、
  喊单密度极高且无论据、带付费群链接

### 通道 D:跨平台迁移监控

- 监控上游平台:Reddit 垂直版块、Substack、雪球/知乎专栏
- 有研究能力的人通常先出现在垂直社区,成名后迁移到大平台
- 对已验证源,监控其跨平台账号关联,任何平台先发都要捕获

### 漏斗汇总

```
四通道进料 → 候选池(candidate)
  → Stage 1 廉价过滤:账号年龄>6月、发文频率适中、非纯转发号
  → Stage 2 LLM 内容分类(通道C规则)
  → Stage 3 历史回填:抓取全部历史内容 → LLM 抽取 PredictionRecord
            → 用历史行情批量验证 → 计算初始 SourceScore
  → Stage 4 Wilson 下界 > 阈值 → 进入 watchlist,开启实时监控
```

## 3. 红旗规则(自动触发,冻结或降权)

| 红旗 | 检测方式 | 含义 |
|---|---|---|
| 删帖 | 定期比对存档与线上 | 删除错误预测=记录造假 |
| 语言/市场突变 | 发文语言、标的市场分布突变 | 角色从研究转向流量变现(如英文美股号突然中文推A股) |
| 重复喊单密集化 | is_repeat_call 频率上升 | 疑似出货期 |
| 自报收益 | LLM 标记 | 不可验证的权威建构 |
| 影响力>研究力反转 | influence 急升 + rolling 研究分下滑 | 进入反身性阶段,信号失效 |
| 付费转化推进 | 出现付费群/课程/订阅推广 | 激励结构改变 |
| 履历不可验证+高调人设 | 人工标记 | 人设型账号特征 |

红旗不等于拉黑,但触发后:该源新预测不计入交易信号,进入隔离观察。

## 4. 实时监控与交易衔接(系统边界)

- watchlist 内源的新内容 → 实时抽取 PredictionRecord → 推送人工审核
- 决策保留人工确认:系统负责"不遗漏、不遗忘、不被情绪骗",最终下单由人拍板
- 多源共振(≥2 个独立 alpha_source 在时间窗内指向同一标的)为最高优先级提醒
- 对 event_mover 源:推文即事件,提醒但默认不跟,除非有事件交易子策略

## 5. MVP 路线图(Claude Code 任务拆解)

### Phase 1:验证管道可行性(1-2 周)
1. 选 1 个测试源(如 Serenity 全部历史推文)
2. 实现:内容抓取 → LLM 抽取 PredictionRecord → 行情验证 → 输出 SourceScore
3. 交付:单源回测报告,人工抽查 LLM 抽取准确率(目标 >90%)
4. 关键检验:用 captured_at 模拟延迟后,跟单收益是否仍为正?
   (预期答案:早期为正,近期被定价 → 验证 regime 模型)

### Phase 2:历史回填 + 记分牌(2-4 周)
1. 人工挑 15-20 个源(你已有名单 + deepvan 等),全量历史回填
2. 实现 SourceScore 计算、Wilson 下界排序、rolling 衰减检测
3. 交付:第一版记分牌 dashboard

### Phase 3:发现管道(4-8 周)
1. 通道 A 异动筛选器 + 历史反查(优先,产出最直接)
2. 通道 B 图谱挖掘
3. 漏斗自动化:candidate → watchlist 流转

### Phase 4:实时化
1. watchlist 实时监控 + 新预测推送
2. 红旗自动检测
3. 多源共振提醒

### 数据源备忘
- 推特:官方 API 贵,评估第三方数据服务或自建采集;高级搜索是通道A的主力工具
- 行情:美股日线免费源够用(yfinance 起步);A股用 akshare/tushare
- 共识预期:免费源有限,Phase 1 可先只做价格验证,量化预测验证后补
- 所有原始内容落库 + 定期快照,存储成本极低,价值极高

## 6. 已知风险与限制

1. **回测幸存者偏差**:通道 A 天然只找到"赌对的人",必须用其后续预测的样本外表现来确认能力,种子记录本身不计入评分。
2. **LLM 抽取错误**:方向/标的抽错会污染评分,Phase 1 必须人工抽查校准 prompt。
3. **推特数据获取是最大工程不确定性**,先评估成本再定采集方案。
4. **法律边界**:跟踪公开言论合法;但若发现源存在操纵市场行为,远离即可,系统数据不构成举报义务也不应卷入。
5. **本系统输出的是研究线索,不是投资建议**;最终决策与风险自担。
